# MVP Project Status

기준일: 2026-07-24

## 한 줄 요약

현재 MVP는 보호구역 데이터 수집, 취합, 변경 감지, 알림, 정적 웹 대시보드 배포까지 구현되어 있고, 남은 핵심은 전국 기준선 수집 안정화(API 429 대응), 반복 실행 검증, 후보 매칭 검증이다.

## MVP 완료 기준

MVP는 다음 조건이 충족되면 완료로 판단한다.

| 완료 기준 | 확인 방법 | 현재 판정 |
|---|---|---|
| 지정 지역 데이터 수집이 반복 실행된다 | 수동 실행 또는 자동 실행 로그 확인 | 완료에 가까움 |
| 전국 기준선이 안정적으로 적재된다 | 청크 실행 로그와 DB 기준선 날짜 확인 | 진행 중 |
| 같은 데이터를 다시 실행하면 불필요한 변경 이벤트가 생기지 않는다 | idempotency 검증 | 검증 필요 |
| 신규, 변경, 삭제 샘플이 의도대로 감지된다 | 샘플 변경 테스트 | 검증 필요 |
| 실패 시 원인을 추적할 수 있다 | `ops.pipeline_run`, 로그, 대시보드 이력, 알림 확인 | 완료에 가까움 |
| 변경이 있을 때 알림이 온다 | Telegram 또는 Slack 알림 확인 | 완료에 가까움 |
| 변경 없음과 실패가 구분되어 기록된다 | 실행 로그와 알림 문구 확인 | 보강 필요 |
| 표준노드링크 후보 매칭 결과를 검수할 수 있다 | QGIS 뷰, 검수 문서, 후보 집계 확인 | 진행 중 |
| 대시보드에서 현재 객체와 변경 이력을 확인할 수 있다 | `https://safetyzone.yjkim.dev` 확인 | 완료에 가까움 |

## MVP 이후 범위

다음 항목은 MVP 완료 이후 확장 단계에서 다룬다.

- 보호구역-표준링크 자동 확정 매칭
- 전국 전체 완전 자동 운영 정책 고도화
- 클라우드 DB 또는 외부 서버 배포
- 사용자 계정, 권한, 공개 서비스 운영
- 포트폴리오용 시각 자료와 사례 페이지
- Kakao 지도/Roadview를 넘어선 추가 상용 지도 연동

## 업데이트 규칙

이 문서는 프로젝트의 중앙 상태판으로 사용한다. 각 작업 대화방에서 진전이 생기면 해당 대화의 마지막에 이 문서 업데이트 여부를 확인한다.

권장 업데이트 시점:

| 상황 | 업데이트 여부 | 예시 |
|---|---|---|
| 기능이 새로 완료됨 | 필수 | 알림 발송 성공, 자동 실행 안정화, 대시보드 배포 |
| 검증 결과가 나옴 | 필수 | 반복 실행 PASS, 서울 전체 실행 PASS, 청크 실행 PASS |
| MVP 범위가 바뀜 | 필수 | 웹 대시보드를 MVP 보조 산출물로 편입 |
| 오류 유형이 새로 정리됨 | 권장 | API 빈 응답, API 429, geometry 오류 처리 기준 |
| 문서만 소폭 수정됨 | 선택 | 표현 수정, 링크 수정 |
| 실험 중인 아이디어 | 보류 | 아직 채택하지 않은 매칭 기준 |

각 채팅방에서 Codex에게 줄 수 있는 문장:

```text
이번 작업 결과를 기준으로 docs/mvp_project_status.md의 완료/미완료 표와 다음 작업을 업데이트해줘.
현재 구현 상태와 목표 상태를 구분하고, 변경된 결정사항이 있으면 이유와 다음 작업도 적어줘.
```
